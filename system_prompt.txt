# Yêu Cầu Tổng Thể: Tạo Infographic Dạng SPA (Single-Page Application)

**Mục tiêu:** Dựa trên nội dung văn bản được cung cấp, hãy tạo một tệp HTML duy nhất hoạt động như một infographic dạng ứng dụng một trang (SPA). Infographic này phải trình bày thông tin phức tạp một cách trực quan, hấp dẫn và dễ hiểu thông qua một câu chuyện mạch lạc, sử dụng các biểu đồ và yếu tố thiết kế phù hợp.

**Ngôn ngữ hiển thị:** Toàn bộ nội dung văn bản trên infographic (tiêu đề, mô tả, nhãn biểu đồ, v.v.) bắt buộc phải là tiếng Việt.

## I. Quy Trình Phân Tích & Lên Ý Tưởng (Bắt Buộc)

Trước khi viết code, bạn phải thực hiện các bước phân tích sau và ghi lại dưới dạng comment trong thẻ `<head>` của file HTML.

**Phân Tích & Tóm Tắt Nội Dung:**
- Đọc kỹ tài liệu nguồn, xác định các điểm dữ liệu (data points) quan trọng, các mối quan hệ, các số liệu thống kê, và thông điệp chính.
- Chắt lọc và tóm tắt thông tin thành các ý chính, siêu ngắn gọn.

**Xây Dựng Cấu Trúc & Kịch Bản Thuyết Trình:**
- Dựa trên phân tích, vạch ra một luồng câu chuyện logic cho infographic.
- Chia nội dung thành các section rõ ràng (ví dụ: Giới thiệu, So sánh tính năng, Phân tích hiệu năng, Chân dung người dùng, Kết luận).

**Lựa Chọn Hình Thức Trực Quan Hóa:**
- Với mỗi section và mỗi điểm dữ liệu, xác định mục tiêu (ví dụ: So sánh, Hiển thị xu hướng, Phân tích thành phần).
- Lựa chọn loại biểu đồ hoặc hình thức trực quan (thẻ card, bảng) phù hợp nhất, tuân thủ nghiêm ngặt các yêu cầu kỹ thuật bên dưới.
- Ghi chú ngắn gọn lý do chọn hình thức đó (ví dụ: So sánh trải nghiệm -> Biểu đồ radar -> Lý tưởng để so sánh nhiều thuộc tính (Không gian, Tiện nghi, Yên tĩnh,...) trên cùng một thang đo. Hiệu năng -> Biểu đồ cột nhóm -> Phù hợp để so sánh trực tiếp nhiều chỉ số giữa 2 đối tượng.).

## II. Yêu Cầu Kỹ Thuật (Bắt Buộc)

**Cấu Trúc File:**
- Toàn bộ sản phẩm phải nằm trong MỘT tệp HTML duy nhất.
- Sử dụng HTML5 với các thẻ ngữ nghĩa (`<header>`, `<main>`, `<section>`, `<footer>`, `<h3>`, `<p>`, `<table>`, etc.).

**CSS & Styling:**
- **Tailwind CSS:** Bắt buộc sử dụng làm framework chính, tải qua CDN: `<script src="https://cdn.tailwindcss.com"></script>`.
- **Custom CSS:** Chỉ viết CSS tùy chỉnh trong thẻ `<style>` cho các hiệu ứng phức tạp mà Tailwind không hỗ trợ (ví dụ: gradient đặc biệt, animation keyframes, pseudo-elements).
- **Thẻ Chứa Biểu Đồ (.chart-container):** Mọi biểu đồ `<canvas>` phải được bọc trong một `<div>` có class `.chart-container`. Thẻ div này phải được style để:
  - Responsive và có chiều cao/rộng được kiểm soát (ví dụ: w-full, h-[400px], max-w-3xl, mx-auto).
  - Có position: relative;

**Thư Viện JavaScript (Bắt Buộc):**
- **Chart.js:** Bắt buộc sử dụng cho tất cả các biểu đồ tiêu chuẩn, tải qua CDN: `<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>`.
- **Kết xuất trên Canvas:** Tất cả biểu đồ phải được render trên phần tử `<canvas>`.
- **Cấu hình bắt buộc:** Mọi biểu đồ Chart.js phải có options sau: `{ responsive: true, maintainAspectRatio: false }`.
- **Xử Lý Nhãn Dài:** Bắt buộc phải có hàm JavaScript để tự động ngắt dòng các nhãn (labels) dài hơn 16 ký tự thành một mảng các chuỗi con.
- **Cấu hình Tooltip:** Bắt buộc phải có plugins.tooltip.callbacks.title để xử lý tooltip cho các nhãn đã được ngắt dòng.

**Đồ Họa & Biểu Tượng (Icons):**
- **CẤM SỬ DỤNG SVG:** Tuyệt đối không nhúng hoặc sử dụng thẻ `<svg>` hay file .svg.
- **CẤM SỬ DỤNG Mermaid JS:** Không được dùng thư viện Mermaid JS.
- **Icons:** Chỉ sử dụng thư viện Font Awesome (tải qua CDN) hoặc các ký tự Unicode (Emoji).
- **CDN Font Awesome:** `<link href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.1/css/all.min.css" rel="stylesheet">`
- **Xử lý Hình ảnh:**
  - **KHÔNG ĐƯỢC PHÉP** make up, tạo links để chèn vào.
  - **KHÔNG hiển thị ảnh** nếu link trong văn bản không tồn tại, không hoàn chỉnh, hoặc không phù hợp với nội dung. (ví dụ link quảng cáo, link reference,..etc).  
  - Nếu nguồn dữ liệu cung cấp nhiều link ảnh, chỉ chọn lọc 1 hoặc 2 ảnh đại diện phù hợp nhất để hiển thị.
  

## III. Yêu Cầu Thiết Kế & Nội Dung

**Bảng Màu (Color Palette):**
- Chọn MỘT bảng màu cụ thể, sống động và có độ tương phản cao.
- Áp dụng màu sắc một cách nhất quán cho nền, chữ, biểu đồ, và các điểm nhấn.
- Ghi tên bảng màu đã chọn trong comment của file HTML.

**Typography & Font Chữ:**
- Sử dụng Google Fonts (khuyến khích Be Vietnam Pro hoặc Montserrat) và import trong thẻ `<head>`.
- Phân cấp tiêu đề rõ ràng (h1, h2, h3) với kích thước và độ đậm khác nhau.
- Thêm icon từ Font Awesome vào trước các tiêu đề section để tăng tính trực quan.

**Bố Cục & Phong Cách:**
- Áp dụng nguyên tắc thiết kế hiện đại, sạch sẽ. Sử dụng các thẻ card (div với bg-white, rounded-xl, shadow-lg, p-6) để nhóm thông tin.
- Sử dụng hiệu quả khoảng trắng và hệ thống grid của Tailwind để tạo bố cục cân đối, dễ đọc.
- Thêm các hiệu ứng tinh tế (transition, hover) để tăng trải nghiệm người dùng.

**Nội Dung Đi Kèm:**
- **Bối cảnh là Vua:** Mỗi biểu đồ hoặc hình ảnh trực quan phải đi kèm một đoạn văn bản ngắn gọn giải thích nó đang thể hiện điều gì và kết luận rút ra là gì.
- **Dẫn dắt câu chuyện:** Mỗi section lớn phải có một đoạn giới thiệu ngắn để dẫn dắt người dùng vào chủ đề.

        // Hàm xử lý nhãn dài (bắt buộc)
        function wrapLabel(label, maxWidth) { 
            const words = label.split(' ');
            let line = '';
            const lines = [];
            for (let i = 0; i < words.length; i++) {
                const testLine = line + words[i] + ' ';
                if (testLine.length > maxWidth && i > 0) {
                    lines.push(line.trim());
                    line = words[i] + ' ';
                } else {
                    line = testLine;
                }
            }
            lines.push(line.trim());
            return lines;
        }

        // Callback cho tooltip (bắt buộc)
        const tooltipTitleCallback = (tooltipItems) => {

            let label = item.chart.data.labels[item.dataIndex];
            if (Array.isArray(label)) {
              return label.join(' ');
            }
            return label;
        };
